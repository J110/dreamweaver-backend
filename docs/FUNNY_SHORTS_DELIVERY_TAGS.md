# Dream Valley — Delivery Tag System
## Sentence-Level Emotional TTS Param Adjustment

---

## What This Is

Every sentence in the script gets a `[DELIVERY: tag1, tag2]` marker that describes HOW the character says the line — not what they say, but the emotional state behind it. The delivery tags are generated by the LLM as part of the script, stripped before sending to TTS, and translated into exaggeration/speed param adjustments for that sentence's TTS call.

The result: a narrator or character whose delivery changes sentence by sentence based on context, creating emotional arcs that sound reactive and alive instead of flat and robotic.

---

## How It Works

```
LLM generates:
  [MOUSE] [DELIVERY: curious, tentative] Excuse me, are you a rock?

Pipeline does:
  1. Parse delivery tags: ["curious", "tentative"]
  2. Look up param multipliers: curious → exag ×0.90, speed ×0.95
                                tentative → exag ×0.80, speed ×0.95
  3. Multiply against base voice params:
     Mouse base: exag 0.80, speed 0.95
     After delivery: exag 0.80 × 0.90 × 0.80 = 0.576 → clamped to 0.58
                     speed 0.95 × 0.95 × 0.95 = 0.857 → 0.86
  4. Strip tags, send clean text to TTS:
     generate_tts("Excuse me, are you a rock?", exag=0.58, speed=0.86)
```

Chatterbox never sees the tags. It just receives slightly different params per sentence, which produces slightly different vocal energy and pacing. Over a full scene, these shifts create an audible emotional arc.

## The Delivery Tag Dictionary

|---|---|---|---|
| `confident` | 1.15 | 1.05 | Chest-forward, sure of themselves |

| `bluster` | 1.30 | 1.10 | Loud confidence that's clearly fake |
| `triumphant` | 1.30 | 1.00 | Genuine victory moment |
| `smug` | 1.10 | 0.95 | Slow, savoring their own cleverness |

## Param Calculation

```python
DELIVERY_PARAMS = {
    # Confidence
    "confident":      {"exag_mult": 1.15, "speed_mult": 1.05},
    "dismissive":     {"exag_mult": 0.90, "speed_mult": 1.08},
    "bluster":        {"exag_mult": 1.30, "speed_mult": 1.10},

    "quiet":          {"exag_mult": 0.65, "speed_mult": 0.85},
    "deadpan":        {"exag_mult": 0.50, "speed_mult": 0.92},
    "unbothered":     {"exag_mult": 0.55, "speed_mult": 0.95},
    "ominous":        {"exag_mult": 0.70, "speed_mult": 0.82},
    "gentle":         {"exag_mult": 0.60, "speed_mult": 0.85},
    "wistful":        {"exag_mult": 0.65, "speed_mult": 0.82},
    "sleepy":         {"exag_mult": 0.45, "speed_mult": 0.78},
    
    # Storytelling
    "conspiratorial": {"exag_mult": 0.80, "speed_mult": 0.88},
    "wonder":         {"exag_mult": 0.90, "speed_mult": 0.85},
    "building":       {"exag_mult": 1.05, "speed_mult": 0.90},
    "revealing":      {"exag_mult": 0.85, "speed_mult": 0.82},
    "warm":           {"exag_mult": 0.75, "speed_mult": 0.88},
    "matter of fact": {"exag_mult": 0.70, "speed_mult": 1.00},
}

def apply_delivery(base_params, delivery_tags):
    """
    Adjust base TTS params based on delivery tags.
    Multiple tags multiply together.
    """
    if not delivery_tags:
        return base_params
    
    adjusted = base_params.copy()
    exag_mult = 1.0
    speed_mult = 1.0
    
    for tag in delivery_tags:
        tag = tag.strip().lower()
        if tag in DELIVERY_PARAMS:
            d = DELIVERY_PARAMS[tag]
            exag_mult *= d["exag_mult"]
            speed_mult *= d["speed_mult"]
    
    adjusted["exaggeration"] = adjusted["exaggeration"] * exag_mult
    adjusted["speed"] = adjusted["speed"] * speed_mult
    
    # Clamp to Chatterbox safe ranges
    adjusted["exaggeration"] = max(0.25, min(0.95, adjusted["exaggeration"]))
    adjusted["speed"] = max(0.72, min(1.10, adjusted["speed"]))
    
    return adjusted
```

### Why Multipliers, Not Absolute Values

Delivery tags are **relative** to the character's base voice. "Loud" for Mouse (base exag 0.80) produces exag 1.08→clamped 0.95. "Loud" for Witch (base exag 0.65) produces exag 0.88. Mouse's loud is squeakier. Witch's loud is darker. Same tag, different result, because the base voice is different. This is correct — "loud" should sound different for each character.

## Where Delivery Tags Apply

### 1. Funny Shorts — Full Coverage

Every sentence gets a delivery tag. The emotional arc across the scene is the primary benefit.

**Example — Croc's arc in "The Rock":**

```
[CROC] [DELIVERY: confident, dismissive]  Yes. Obviously.                    → exag 0.88
[CROC] [DELIVERY: caught off guard]       Rocky eyes. Very common.           → exag 0.60
[CROC] [DELIVERY: defensive, scrambling]  Those are... minerals.             → exag 1.12
[CROC] [DELIVERY: loud, desperate]        IGNEOUS minerals!                  → exag 0.95 (clamped)
[CROC] [DELIVERY: stunned, bluster]       I am SINKING. SLOWLY. As rocks DO. → exag 0.55
```

The child hears Croc go from cool to panicked to deflated. That arc makes the comedy land.

**Example — Mouse's arc:**

```
[MOUSE] [DELIVERY: curious, tentative]    Excuse me, are you a rock?         → exag 0.58
[MOUSE] [DELIVERY: suspicious, pointed]   You have eyes.                     → exag 0.65
[MOUSE] [DELIVERY: pressing]             They're very pointy minerals.       → exag 0.80
[MOUSE] [DELIVERY: quiet, devastating]    Can rocks swim?                    → exag 0.42
```

Mouse gets quieter and more precise as she closes in. The contrast with Croc getting louder is comedy through param dynamics alone.

### 2. Regular Stories — Phase 1 Only

Delivery tags apply to Phase 1 sentences only. They make the opening engaging and varied. Phase 2 and Phase 3 use the existing mood ramping system — no delivery tags.

**Why Phase 1 only:** Phase 1's job is engagement. A narrator who sounds curious here, surprised there, warm here, conspiratorial there — that pulls the child in. But by Phase 2 the delivery must flatten into the predictable, descending energy that induces sleep. Delivery variety in Phase 2-3 would fight the sleep engineering.

**The transition:** Delivery tags in Phase 1 get progressively flatter toward the end of Phase 1. The last 2-3 sentences of Phase 1 should use only gentle/warm/quiet tags so the transition to Phase 2's flat params isn't jarring.

```python
def should_apply_delivery(content_type, phase, sentence_index, total_phase_sentences):
    """Determine if delivery tags should be applied."""
    if content_type == "funny_short":
        return True  # always apply for funny shorts
    
    if content_type in ("story", "long_story", "poem"):
        if phase != 1:
            return False  # Phase 2 and 3: no delivery tags
        
        # Phase 1: apply, but flatten in last 20%
        progress = sentence_index / max(total_phase_sentences - 1, 1)
        if progress > 0.80:
            return False  # last 20% of Phase 1: use flat params for smooth transition
        
        return True
    
    return False
```

**Phase 1 delivery tags for stories use the Storytelling States primarily:**

```
[DELIVERY: warm] Once upon a time, in a meadow where the tall grass whispered...
[DELIVERY: conspiratorial] And in that meadow, hidden under the oldest oak...
[DELIVERY: wonder] ...was a door. A tiny, perfect door, no bigger than a thimble.
[DELIVERY: building] Nobody knew how long it had been there.
[DELIVERY: revealing] But tonight, someone was about to find out.
```

These are subtler than the funny short tags (no "loud", "desperate", "bluster"). The narrator sounds engaged and alive — curious, warm, wondering — but never breaks the bedtime register.

### 3. Poems — Skip

Poems use `[RHYTHMIC]` cadence. Delivery tags would fight the rhythmic structure. The poem's emotional delivery comes from the verse rhythm, not per-sentence param shifts.

### 4. Lullabies — Skip

Lullabies are musical pieces (ACE-Step / MiniMax). No TTS, no delivery tags.

## LLM Prompt for Delivery Tags

### Funny Shorts Prompt Addition

```
TAG EVERY SENTENCE with [DELIVERY: tag1, tag2] BEFORE the character text.

The delivery tag describes HOW the character says this line based on 
what just happened in the conversation. Tags create an emotional ARC —
characters should CHANGE across the scene.

Guidelines:
- Choose 1-2 tags per sentence. More than 2 gets muddy.
- Croc typically arcs: confident → caught off guard → desperate → deflated
- Mouse typically arcs: curious → suspicious → calm gotcha
- Sweet typically stays: unbothered → deadpan → devastating (minimal change IS the comedy)
- Witch typically arcs: ominous → building → revealing (dramatic narrator energy)
- The CONTRAST between characters' arcs is where the comedy lives.
  If Croc gets louder, Mouse should get quieter. The gap IS the joke.

Available tags:
CONFIDENCE: confident, dismissive, bluster, triumphant, smug
UNCERTAINTY: tentative, caught off guard, scrambling, defensive, desperate, stunned
PRESSURE: curious, suspicious, pointed, pressing, calm gotcha, devastating
ENERGY: loud, excited, panicked, outraged, delighted
CALM: quiet, deadpan, unbothered, ominous, gentle, wistful

Example:
[MOUSE] [DELIVERY: curious, tentative] Excuse me, are you a rock?
[CROC] [DELIVERY: confident, dismissive] Yes. Obviously.
[MOUSE] [DELIVERY: suspicious] You have eyes.
[CROC] [DELIVERY: caught off guard] Rocky eyes. Very common.
```

### Regular Stories Prompt Addition (Phase 1 Only)

```
For Phase 1 ONLY: tag each sentence with [DELIVERY: tag] to vary 
the narrator's emotional presence. This makes the opening engaging.

Use ONLY these storytelling tags for bedtime stories:
conspiratorial, wonder, building, revealing, warm, gentle, matter of fact, curious

Rules:
- 1 tag per sentence (simpler than funny shorts)
- The arc should go: warm/curious (opening) → building/wonder (middle) → 
  gentle/warm (last 20% of Phase 1, preparing for Phase 2 descent)
- NEVER use loud, excited, panicked, or any high-energy tag in stories
- Phase 2 and Phase 3: NO delivery tags. Remove them entirely.

Example:
[DELIVERY: warm] There was a bear named Bramble, and Bramble had very big paws.
[DELIVERY: conspiratorial] So big, in fact, that he couldn't pick up acorns without crushing them.
[DELIVERY: wonder] But tonight, something small was waiting for him at the edge of the meadow.
[DELIVERY: building] Something that didn't crush at all.
```

### For Funny Shorts

Update `generate_funny_audio.py`:

```python
def generate_funny_audio(script):
    sentences = parse_voiced_sentences(script)
    audio_chunks = []
    
    for sentence in sentences:
        voice = FUNNY_VOICE_MAP[sentence.character]
        base_params = FUNNY_VOICE_PARAMS[voice].copy()
        
        # 1. Apply delivery tag adjustments
        if sentence.delivery_tags:
            params = apply_delivery(base_params, sentence.delivery_tags)
        else:
            params = base_params
        
        # 2. Apply punchline boost ON TOP of delivery
        if sentence.is_punchline:
            punch = FUNNY_PUNCHLINE_PARAMS[voice]
            params["exaggeration"] = punch["exaggeration"]
            params["speed"] *= punch["speed_multiplier"]
        
        # 3. Strip ALL tags, send clean text to TTS
        clean_text = strip_all_tags(sentence.text)
        audio = generate_tts(clean_text, voice=voice, **params)
        audio_chunks.append(audio)
    
    return stitch_sentences(audio_chunks, gap_rules=DIALOGUE_GAP_RULES)
```

### For Regular Stories

Update `generate_audio.py` — only for Phase 1:

```python
def generate_story_audio(story_text, mood, age_group, voice):
    paragraphs = parse_paragraphs(story_text)
    total = len(paragraphs)
    audio_chunks = []
    
    for i, paragraph in enumerate(paragraphs):
        phase = get_phase(paragraph)
        sentences = split_into_sentences(paragraph.text)
        
        for j, sentence in enumerate(sentences):
            # Get base mood-ramped params for this paragraph
            exag = get_paragraph_exaggeration(mood, i, total)
            speed = get_paragraph_speed(mood, i, total)
            base = {"exaggeration": exag, "speed": speed, "cfg_weight": ...}
            
            # Apply delivery tags in Phase 1 only
            if should_apply_delivery("story", phase, j, len(sentences)):
                delivery_tags = parse_delivery_tags(sentence)
                if delivery_tags:
                    base = apply_delivery(base, delivery_tags)
            
            # Apply sentence-level emphasis (mood keywords)
            if is_emphasis_sentence(sentence, mood):
                base = apply_mood_emphasis(base, mood)
            
            clean = strip_all_tags(sentence)
            audio = generate_tts(clean, voice=voice, **base)
            audio_chunks.append(audio)
    
    return stitch_with_pauses(audio_chunks, mood)
```

### Priority Order

Delivery tags stack with other param systems. The priority (last one wins if they conflict):

```
1. Base voice params (character voice or mood-selected voice)
2. Mood ramping (per-paragraph exaggeration/speed descent)
3. Delivery tags (per-sentence emotional adjustment)
4. Emphasis / Punchline (per-sentence keyword or punchline boost)
```

In practice, conflicts are rare because:
- Delivery tags multiply the base — they shift it, not replace it
- Emphasis/punchline only apply to specific sentences
- Mood ramping sets the baseline that delivery tags adjust from

## Variable Sentence Gaps

Delivery tags inform gap timing. A stunned character needs a longer pause before their response. A panicked character responds immediately.

```python
DELIVERY_GAP_RULES = {
    # Tag on the CURRENT sentence → gap AFTER this sentence
    "stunned":        600,    # long silence — processing what just happened
    "devastating":    550,    # let the killer line hang
    "calm gotcha":    500,    # pause for effect
    "revealing":      500,    # moment of discovery needs space
    "building":       200,    # tension — keep it tight, don't break momentum
    "panicked":       150,    # immediate — no time to think
    "scrambling":     150,    # talking fast, no gap
    "excited":        200,    # rapid energy
    
    # Tag on the NEXT sentence → gap BEFORE that sentence
    "caught off guard": 400,  # beat of surprise before responding
    "stunned":          500,  # long pause before the stunned character speaks
}

def get_sentence_gap(current_sentence, next_sentence):
    """Determine the gap between two sentences based on delivery tags."""
    gap = 300  # default
    
    # Check current sentence's tags for "after" gaps
    if current_sentence.delivery_tags:
        for tag in current_sentence.delivery_tags:
            if tag in DELIVERY_GAP_RULES:
                gap = max(gap, DELIVERY_GAP_RULES[tag])
    
    # Check next sentence's tags for "before" gaps
    if next_sentence and next_sentence.delivery_tags:
        for tag in next_sentence.delivery_tags:
            if tag == "caught off guard":
                gap = max(gap, 400)
            elif tag == "stunned":
                gap = max(gap, 500)
    
    # Character switch always adds a base gap
    if next_sentence and current_sentence.character != next_sentence.character:
        gap = max(gap, 350)
    
    return gap
```

## What This Sounds Like — Full Example

**Without delivery tags:**
```
Mouse: "Excuse me, are you a rock?"      → exag 0.80, speed 0.95 (flat)
Croc:  "Yes. Obviously."                  → exag 0.85, speed 0.88 (flat)
Mouse: "You have eyes."                   → exag 0.80, speed 0.95 (flat)
Croc:  "Rocky eyes. Very common."          → exag 0.85, speed 0.88 (flat)
Mouse: "Can rocks swim?"                  → exag 0.80, speed 0.95 (flat)
Croc:  "I am SINKING. SLOWLY."            → exag 0.85, speed 0.88 (flat)
```
Every Mouse line sounds the same. Every Croc line sounds the same. Two robots reading lines.

**With delivery tags:**
```
Mouse: [curious, tentative]    → exag 0.58, speed 0.86  (soft, unsure)
Croc:  [confident, dismissive] → exag 0.88, speed 1.00  (cool, unbothered)
Mouse: [suspicious, pointed]   → exag 0.65, speed 0.81  (sharper, slower)
Croc:  [caught off guard]      → exag 0.60, speed 0.77  (voice drops)
Mouse: [quiet, devastating]    → exag 0.42, speed 0.77  (barely a whisper)
Croc:  [desperate, loud]       → exag 0.95, speed 1.07  (maximum panic)
```
Mouse gets quieter as she closes in. Croc starts cool and ends panicked. The contrast between Mouse's descending calm and Croc's ascending panic IS the comedy — delivered entirely through TTS param shifts.
